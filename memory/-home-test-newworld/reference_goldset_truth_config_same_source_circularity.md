**BL-60 分类金标（2026-07-14）踩到的两个金标设计陷阱，都是蓝军复核逮出来的，纯读报告看不出来。**

## 1. ★ 真值与被评测配置同源 = 循环论证（最隐蔽）

Owner 拍板「真值 = 标题 + 源站 tags」，标注员据此标；同时评测的配置 B 也是「把 tags 喂给 LLM」。
→ **两端看同一份信号**。实测 19.3% 的 evidence 只引 tags（不引标题）——这些标签的真值**就是 tag 本身**。

后果：tag 的噪声在**标注端和模型端同向传播**，F1 完全看不见它。B 相对 A（只看标题）的增益，
**无法区分「模型看到了真相」与「模型看到了标注员看到的同一份噪声」**。

而项目自己的铁律「任何『见 tag X 即判 Y』必先量化该 tag 精确率」在此口径下**结构性不可执行**
——拿什么量化？金标就是用 tag 标的。

**判据**：设计金标时问一句「**我的真值信号，和我要评测的配置的输入信号，是不是同一套？**」
是 → 该配置的分数只是「与知情标注的一致性上界」，不是准确率。
**破法**：抽 50~100 条「只靠 X 信号标出来的」样本，用**独立于 X 的信号**（如视觉/人工）做一次性校准审计，
量化 X 型真值的噪声率，再把上界修正成真实值。这是校准，不是给全集重标，成本可控。

## 2. 多标签 macro F1 的类集必须两榜固定一致

原报告「macro F1 +21%」是**假的**：两行 macro 的**类集不同**——
- 全金标榜：4 个结构性不覆盖类（support=2/4/8/2）以 F1≈0 **进了 macro 拖底**
- Tier B 随机榜：同样 4 类 support=0 **被自动排除**

→ 纵向对比混入了「类集差异」，不纯是「加权不同」。修正为**固定 16 类**（显式排除声明不覆盖的类）后，
真实增益是 **+0.05~0.07**，不是 +21%。

**铁律**：报 macro 前先把类集钉死并写进报告；「support=0 自动排除」是隐式类集变化，必须显式化。
声明「某类不覆盖、不评价」就**不能让它进 macro 均值**——否则文档自相矛盾。

## 3. 附带：同模型双实例 ≠ 独立标注员

两名标注员都是 Fable 5 → 共享系统性直觉，正是「自评≠评审」铁律要防的。
且一致性样本里机械规则类（region 推导）占大头，两人执行同一条 if 语句当然一致
→ micro 0.879 大半在测**规则执行**不在测**判断**。
per-class 一致性表**必须带 support 列**（support=1~5 的 F1 没有统计意义，别写进「可直接信」）。
