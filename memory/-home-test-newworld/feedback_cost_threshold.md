用户明确表态（2026-04-20）：**类似 aws-s 月流量费 $50 这种"几百美金一个月"级别的成本，都不用考虑**。

**Why**：项目定位是 500 万 DAU 中期目标的反审查系统。**系统稳定性、用户可达性、反脆弱性 >> 小额基建成本**。为省 $30-$500/月去做架构妥协（如单源、缩编冗余、砍备线）属于**捡芝麻丢西瓜**。典型场景：模式 I 主备切换 vs 模式 II 双源并发，用户选 II（月多 $25-$40 成本）消除 agent SPOF 风险，因为 agent 挂掉 5 min 损失的用户价值远超成本。

**How to apply**：
1. 架构设计时**不为 $50-$500/月节省**砍冗余层、砍备线、砍第三方 SaaS
2. KPI 里**不纳入**百美金级成本告警（除非超过 $1000/月阈值再讨论）
3. 基建选型优先考虑"反脆弱性" / "去中心化" / "SPOF 消除"，**成本是次要因素**
4. 只有**量级跳变**（$1k+/月、$10k+/年）才需要慎重讨论成本
5. 反面案例：不要再出现"aws-s 备线不立即部署等 2 周省 $17" 这类决策（`S_P_ARCH_V3_2_3_FINAL.md §2 ι` 已纠偏过一次）
